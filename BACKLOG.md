## Идеи для развития

### 14. Удаление пользователя администратором
В панели администратора на вкладке «Участники» добавить кнопку удаления пользователя. По нажатию — диалог подтверждения, затем удаление записи пользователя из БД. Активная сессия пользователя должна инвалидироваться — при следующем обращении к защищённому роуту он будет разлогинен. При желании пользователь может зарегистрироваться заново.

### 4. Вход через Telegram
Добавить авторизацию через Telegram (Telegram Login Widget) как альтернативу Google OAuth и email magic link. Если пользователь вошёл через Telegram — не запрашивать Telegram-контакт отдельно, а подставлять его автоматически из данных авторизации и блокировать поле в ContactsForm.
**Статус:** не работает — Telegram не доставляет код авторизации в приложение. Виджет и конфигурация бота корректны, проблема на стороне Telegram OAuth. Оставлено до выяснения.

### 1. Описание проекта
Добавить на главную страницу описание книжного клуба: что это, как работает, как присоединиться.

### 5. Минимальные данные при авторизации
Исследовать, можно ли не собирать email при входе через Google или magic link — чтобы пользователи делились только Telegram-аккаунтом. Требует проработки: NextAuth v5 использует email как идентификатор сессии.

### 8. Загрузка книг пользователями
Возможность для участников предлагать книги в общий список для обсуждения клубом.

## Выполнено

### 3. Статусы книг
Возможность отмечать книги как прочитанные и закрытые для записи. Участники должны видеть, что запись на книгу недоступна.

### 6. Удаление аккаунта пользователем
Возможность для участника самостоятельно удалить свой аккаунт с сайта.

### 7. Редактирование контактов
Возможность для участника редактировать свои данные (имя, Telegram) после регистрации.

### 9. Число страниц и ссылка на файл
Вернуть в карточку книги отображение числа страниц и ссылки на файл с книгой (поля уже есть в Google Sheets — колонки Pages и Link).

### 10. Описание категорий при фильтрации
При выборе категории (тега) показывать пояснение — в чём смысл этой категории. Например, при клике на тег появляется подпись с описанием.

### 11. Счётчик записавшихся на книгу
Показывать на карточке книги (или в другом месте сайта), сколько человек записалось на каждую книгу. Данные брать из Google Sheets (таблица записей). Помогает участникам видеть популярность книг и принимать решение о записи.

### 12. Подвал с информацией о деплое на странице /admin
В нижней части страницы /admin отображать дату и время последнего деплоя (CET), короткий хеш коммита со ссылкой на GitHub и сообщение коммита. Позволяет быстро убедиться, что свежие изменения уже на сайте.

### 13. Снятие пользователя с книги администратором
В панели администратора на вкладке «Участники» рядом с каждой книгой в столбце «Книги» добавить кнопку-крестик. По нажатию — диалог подтверждения с именем пользователя, затем удаление книги из записи пользователя в Google Sheets. Оптимистичное обновление UI. Если у пользователя не осталось книг — запись остаётся (удаление пользователей — отдельная фича).
