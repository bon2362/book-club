## Идеи для развития

### 17. Переезд на домен slowreading.club

Домен куплен на Namecheap, верифицирован в Resend. Осталось:

- [ ] Vercel: Settings → Domains → добавить `slowreading.club`
- [ ] Namecheap: Advanced DNS → добавить запись от Vercel (CNAME или A)
- [ ] Google Cloud Console: добавить `https://slowreading.club` в Authorized JavaScript origins и `https://slowreading.club/api/auth/callback/google` в Authorized redirect URIs
- [ ] Vercel: ENV `NEXTAUTH_URL` → `https://slowreading.club`
- [ ] Обновить `.devcontainer/init-firewall.sh` — добавить `slowreading.club` в allowlist (сделает Claude)


### 21. Оптимизация изображений обложек
Обложки загружаются через обычный `<img>`. Заменить на Next.js компонент `<Image>` для автоматического сжатия, конвертации в WebP, lazy loading и предотвращения layout shift. Потребует добавить домены обложек в `next.config.mjs` в `images.remotePatterns`.

### 22. Кнопка переключения тёмной темы
Тёмный режим работает через `prefers-color-scheme`, но пользователь не может переключить его вручную. Добавить кнопку в хедер (иконка солнца/луны), сохранять выбор в `localStorage`.

### 23. Toast-уведомления об ошибках
Многие ошибки API молча проглатываются или меняют текст кнопки — пользователь может не понять, что что-то пошло не так. Добавить простой toast-компонент (fixed div снизу экрана) для отображения ошибок и успешных действий.

### 24. Пагинация или виртуализация списка книг
Все книги загружаются сразу. При 50+ книгах на мобильном это заметно. Добавить простую пагинацию или `react-virtual` для виртуализации длинного списка.

### 25. Аналитика посещаемости
Нет понимания, кто заходит, откуда, что смотрит. Варианты: Vercel Analytics (уже доступен в аккаунте, включается одной строкой) или self-hosted Umami (GDPR-compliant, бесплатно).

### 4. Вход через Telegram
Добавить авторизацию через Telegram (Telegram Login Widget) как альтернативу Google OAuth и email magic link. Если пользователь вошёл через Telegram — не запрашивать Telegram-контакт отдельно, а подставлять его автоматически из данных авторизации и блокировать поле в ContactsForm.

### 5. Минимальные данные при авторизации
Исследовать, можно ли не собирать email при входе через Google или magic link — чтобы пользователи делились только Telegram-аккаунтом. Требует проработки: NextAuth v5 использует email как идентификатор сессии.

### 8. Загрузка книг пользователями
Возможность для участников предлагать книги в общий список для обсуждения клубом.

## Выполнено

### 26. Email-уведомление организатору при новой записи
`upsertSignup` теперь возвращает `{ isNew, addedBooks }`. В `/api/signup` при добавлении книг отправляется письмо на `ADMIN_EMAIL` через Resend: имя участника, контакт, email, список добавленных книг. Отправка не блокирует ответ (`.catch`).

### 20. Open Graph теги для социального превью
Добавлены `og:title`, `og:description`, `og:siteName`, `og:locale`, `og:type` и Twitter Card в `app/layout.tsx`. `og:image` не добавлен — нет изображения/логотипа клуба. Стоит создать `public/og-image.png` и добавить его.

### 19. Раскрытие описания книги по клику на текст
В карточке книги описание раскрывается/сворачивается кнопками «Читать далее» и «Свернуть». Кнопки сохранены, добавлен также клик по самому тексту описания. Курсор меняется на pointer для подсказки.

### 18. Кнопка «Наверх» на мобильных
Залипающая круглая кнопка ↑ в правом нижнем углу. Видна только на мобильных (`max-width: 768px`). Появляется когда пролистал вниз больше 2 экранов и начал скроллить обратно вверх.

### 1. Описание проекта
Блок с текстом «Я, Евгений, приглашаю вместе читать и обсуждать книги. Если вам что-то приглянулось из списка, записывайтесь — я свяжусь через Telegram, и согласуем формат» между хедером и строкой поиска. Кнопка × справа скрывает блок; выбор сохраняется в куке `about_closed=1` на год.

### 3. Статусы книг
Возможность отмечать книги как прочитанные и закрытые для записи. Участники должны видеть, что запись на книгу недоступна.

### 6. Удаление аккаунта пользователем
Возможность для участника самостоятельно удалить свой аккаунт с сайта.

### 7. Редактирование контактов
Возможность для участника редактировать свои данные (имя, Telegram) после регистрации.

### 9. Число страниц и ссылка на файл
Вернуть в карточку книги отображение числа страниц и ссылки на файл с книгой (поля уже есть в Google Sheets — колонки Pages и Link).

### 10. Описание категорий при фильтрации
При выборе категории (тега) показывать пояснение — в чём смысл этой категории. Например, при клике на тег появляется подпись с описанием.

### 11. Счётчик записавшихся на книгу
Показывать на карточке книги (или в другом месте сайта), сколько человек записалось на каждую книгу. Данные брать из Google Sheets (таблица записей). Помогает участникам видеть популярность книг и принимать решение о записи.

### 12. Подвал с информацией о деплое на странице /admin
В нижней части страницы /admin отображать дату и время последнего деплоя (CET), короткий хеш коммита со ссылкой на GitHub и сообщение коммита. Позволяет быстро убедиться, что свежие изменения уже на сайте.

### 13. Снятие пользователя с книги администратором
В панели администратора на вкладке «Участники» рядом с каждой книгой в столбце «Книги» добавить кнопку-крестик. По нажатию — диалог подтверждения с именем пользователя, затем удаление книги из записи пользователя в Google Sheets. Оптимистичное обновление UI. Если у пользователя не осталось книг — запись остаётся (удаление пользователей — отдельная фича).

### 16. Вход через magic link
Email-поле + кнопка «Получить ссылку на почту» в `AuthModal` под кнопкой Google. Письма отправляются через Resend с `noreply@slowreading.club`. Кастомный HTML-шаблон с брендингом клуба, plain text fallback, тема «Ссылка для входа в книжный клуб». Домен верифицирован в Resend и Google Postmaster Tools.

### 15. Переключение вида списка книг: сетка / таблица
Иконки-переключатели в строке фильтров (справа). По умолчанию — сетка карточек. Табличный вид: компактные строки без обложек и описаний, цветная полоска статуса слева, название + год + автор, теги, страницы + ссылка, счётчик записавшихся, кнопка действия. Hover-подсветка строк. Выбор сохраняется в `localStorage`.

### 14. Удаление пользователя администратором
Кнопка «Удалить» в таблице участников с диалогом подтверждения. Удаляет пользователя из Postgres (каскадно accounts/sessions), помечает строку в Sheets G=TO DELETE + H=yes (is_deleted_by_admin). JWT callback проверяет существование пользователя в DB — при отсутствии возвращает null, что очищает cookie и разлогинивает пользователя на следующем запросе.
