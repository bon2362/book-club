### Процессы разработки (продолжение)

#### 27. .prettierrc в репозитории · `XS`

Prettier настроен только через VSCode (`devcontainer.json`). Добавить `.prettierrc` в корень репо — чтобы форматирование было одинаковым у всех участников и в CI независимо от редактора.

#### 31. Husky + lint-staged · `S`

Pre-commit хук: запускать `eslint --fix` и `prettier --write` только на staged файлах. Не даёт коммитить явно сломанный или неотформатированный код.

#### 32. Валидация env-переменных · `S`

`process.env.X` возвращает `undefined` без предупреждений — ошибка проявляется глубоко в рантайме. Добавить `@t3-oss/env-nextjs` или zod-схему: при отсутствии обязательной переменной сервер падает на старте с понятным сообщением.

#### 34. Мониторинг ошибок (Sentry) · `S`

Продакшен-ошибки видны только если пользователь сообщит. Подключить Sentry (`@sentry/nextjs`) — бесплатный tier достаточен. Даёт трассировку, контекст запроса и алерты на email при необработанных ошибках.

### UI/UX

#### Компактная шапка — объединить элементы · `S`

Шапка занимает слишком много места. Объединить имя пользователя и ссылку на профиль в один элемент (кликабельное имя). Кнопку «Выйти» заменить на иконку с тултипом. Особенно актуально на мобильных.

#### Прочитанные книги: запись открыта, но скрыты по умолчанию · `S`

Книги со статусом «прочитано» не блокируют запись — новые читательские круги могут читать ту же книгу. Но по умолчанию такие книги скрыты в каталоге (сообщество уже освоило их). Добавить переключатель «Показать прочитанные» в фильтрах для тех, кто хочет их видеть.

#### Убрать кнопку «Предложить книгу» из шапки на мобильных · `XS`

На мобильном экране элементы шапки наезжают друг на друга из-за кнопки «Предложить книгу». Скрыть кнопку в шапке на мобильных (`md:hidden`). Точку входа оставить в теле страницы.

#### Залипающая кнопка сабмита в форме «Предложить книгу» · `S`

При скролле длинной формы кнопка «Отправить» уходит за экран. Зафиксировать кнопку внизу экрана при скролле (sticky/fixed), особенно на мобильных.

#### 22. Кнопка переключения тёмной темы · `M`

Тёмный режим работает через `prefers-color-scheme`, но пользователь не может переключить его вручную. Добавить кнопку в хедер (иконка солнца/луны), сохранять выбор в `localStorage`.

#### 23. Toast-уведомления об ошибках · `S`

Многие ошибки API молча проглатываются или меняют текст кнопки — пользователь может не понять, что что-то пошло не так. Добавить простой toast-компонент (fixed div снизу экрана) для отображения ошибок и успешных действий.

#### 24. Пагинация или виртуализация списка книг · `M`

Все книги загружаются сразу. При 50+ книгах на мобильном это заметно. Добавить простую пагинацию или `react-virtual` для виртуализации длинного списка.

### Функциональность

#### Ранжированное голосование за книги · `L`

Пользователь в личном кабинете может расставить книги, на которые записался, по приоритету — от наиболее интересной к наименее. Администратор видит агрегированные приоритеты при разбивке по группам: кому какая книга важнее, чтобы оптимально распределить людей по читательским кругам. UI: drag-and-drop сортировка списка книг в личном кабинете.

#### Разбивка участников по группам (admin) · `L`

Администратор выбирает, как разбить участников, записавшихся на одну книгу, по группам для обсуждения. UI в admin-панели: список книг → участники → ручное распределение по группам (drag-and-drop или выпадающий список). Результат сохраняется и, опционально, рассылается участникам группы по email.

#### Причина отказа заявки на книгу · `S`

Администратор при отклонении заявки может указать причину отказа (текстовое поле в admin-панели). Пользователь получает email с причиной. Затрагивает: `book_submissions` schema (поле `rejectionReason`), API модерации (`/api/admin/submissions/[id]`), email-шаблон отклонения (`emails/`), UI admin-панели.

#### Удаление книги из каталога администратором · `M`

Администратор может удалить книгу из каталога (из Google Sheets или из `book_submissions`). Особенно нужно для очистки тестовых записей. Затрагивает: новый API-эндпоинт удаления, кнопка в admin-панели с диалогом подтверждения. Сложнее для книг из Sheets — нужно использовать Sheets API для удаления строки.

#### Колонка «Why for club» в Google Sheets · `XS`

Добавить в таблицу Google Sheets новую колонку «Why for club» — текст для блока «Почему стоит прочитать» для книг, добавленных администратором (не через заявку). После добавления колонки — считывать её в `lib/sheets.ts` и передавать в карточку книги (связано с идеей выше о показе этого поля в UI).

#### Поле «Почему стоит прочитать» на карточке книги · `S`

Добавить в карточку книги отображение поля «Почему стоит прочитать» — оно уже есть в `book_submissions` для пользовательских предложений. Для книг из Google Sheets требует колонку «Why for club» (см. выше). Показывать в раскрывающейся секции вместе с описанием.

#### Саммари книги от участников · `L`

Авторизованный участник может написать короткое саммари (личные впечатления, выводы) по книге, которую клуб уже прочитал. Саммари привязано к книге и пользователю, отображается на странице книги или в карточке. Несколько участников могут оставить саммари по одной книге.

Затрагивает: новая таблица в `lib/db/schema.ts` (book_summaries: bookId, userId, text, createdAt), новый API-эндпоинт, UI на карточке/странице книги.

#### Умная фильтрация по автору — поиск по вхождению · `S`

Сейчас «Wolfgang Streeck» и «Wolfgang Streeck, Armin Schäfer» считаются разными авторами в фильтре, потому что сравнивается строка целиком. Нужно разбивать поле «Автор» на отдельных авторов (split по запятой/«и»/«&») и строить фильтр по уникальным именам — тогда выбор «Wolfgang Streeck» покажет все книги, где он соавтор.

#### Предзаполнение формы «Предложить книгу» из активных фильтров · `S`

Если пользователь отфильтровал каталог по теме или автору, а затем открывает форму «Предложить книгу» — поля «Тема» и/или «Автор» предзаполняются значениями активных фильтров. Затрагивает: передачу состояния фильтров в момент открытия формы.

#### Кнопка «Предложить книгу» в табличном виде · `S`

В табличном отображении списка книг отсутствует псевдо-карточка «Предложить книгу» (она есть только в сеточном виде). Добавить кнопку-строку или отдельный элемент в таблице для входа в флоу предложения книги.

#### Фильтр «Мои книги» · `M`

Авторизованный пользователь может отфильтровать каталог, чтобы видеть только книги, на которые он уже записался. Кнопка/переключатель в строке фильтров. Данные о записях уже есть в Google Sheets — нужно сопоставить с текущей сессией пользователя (новый API-эндпоинт или расширение существующего).

#### Ярлык «Новая» + фильтр по новинкам · `M`

Помечать новые книги визуальным ярлыком (badge «Новая») на карточке и добавить фильтр в каталог для просмотра только новинок. Критерий новизны: книга добавлена за последние N дней (из Google Sheets или через заявку) — либо администратор помечает вручную.

Затрагивает: `components/nd/BookCard.tsx` (badge), `app/new-design/page.tsx` (UI фильтра), `lib/sheets.ts` / `lib/books-with-covers.ts` (дата добавления), возможно `lib/db/schema.ts` (дата для книг из заявок).

#### Форма «Предложить книгу» не открывается автоматически после OAuth-редиректа · `M`

Флоу «Предложить книгу → авторизоваться через Google → вернуться» не открывает форму автоматически. Проверить, сохраняется ли `submitIntent` в `callbackUrl` после OAuth-редиректа. Вероятная причина: `callbackUrl` теряется при редиректе через accounts.google.com. Альтернатива: сохранять намерение в `localStorage` перед редиректом.

#### 8. Загрузка книг пользователями

Возможность для участников предлагать книги в общий список для обсуждения клубом.

#### Личный кабинет пользователя · `M`

Отдельная страница (или секция в модалке) для авторизованного пользователя с двумя блоками:

- **Мои предложения** — список книг, которые пользователь предлагал: название, автор, текущий статус (на рассмотрении / одобрена / отклонена). Данные из `book_submissions` по `userId`. Опционально: кнопка отозвать заявку (если `pending`).
- **Мои контакты** — форма редактирования контактной информации (имя, Telegram): то, что сейчас спрашивается при первом входе в `ContactsForm`. Позволяет пользователю обновить данные в любой момент без обращения к администратору.

### 16. Вход через magic link

Кастомный HTML-шаблон с брендингом клуба, plain text fallback, тема «Ссылка для входа в книжный клуб». Домен верифицирован в Resend и Google Postmaster Tools.

Выбор сохраняется в `localStorage`.

### 14. Удаление пользователя администратором

Кнопка «Удалить» в таблице участников с диалогом подтверждения. Удаляет пользователя из Postgres (каскадно accounts/sessions), помечает строку в Sheets G=TO DELETE + H=yes (is_deleted_by_admin). JWT callback проверяет существование пользователя в DB — при отсутствии возвращает null, что очищает cookie и разлогинивает пользователя на следующем запросе.
