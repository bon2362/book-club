## Session Overview

**Topic:** Как показать влияние личных предпочтений на качество распределения читательских групп.

**Goals:** Найти простой интерфейсный сигнал или интерактивную механику, которая помогает участнику увидеть, что небольшое изменение его списка может улучшить общее распределение: больше людей получают более желанные книги, а не просто “все заняты”.

### Session Setup

Контекст: стандартная группа из трёх человек, один человек участвует только в одной группе. Текущий расклад может покрывать всех участников, но быть хуже по суммарной удовлетворённости, чем альтернативный расклад, если один участник добавит книгу или поменяет приоритеты.

## Technique Selection

**Approach:** Progressive Technique Flow

**Progressive Techniques:**

- **Phase 1 - Exploration:** Analogical Thinking для поиска простых метафор влияния.
- **Phase 2 - Pattern Recognition:** Ecosystem Thinking для понимания “узких мест” и взаимосвязей между участниками.
- **Phase 3 - Development:** Solution Matrix для сравнения UX-концепций по понятности, сложности и риску перегруза.
- **Phase 4 - Action Planning:** Decision Tree Mapping для превращения выбранной концепции в состояния интерфейса.

**Journey Rationale:** Задача не в том, чтобы объяснить пользователю алгоритм матчинга, а в том, чтобы показать последствия личного действия через короткий визуальный сигнал или интерактивный момент.

## Phase 1 - Analogical Thinking

Самые перспективные метафоры:

- **Открыть дверь:** личное действие открывает лучший расклад, которого сейчас нет.
- **Командный пас:** участник делает ход, который улучшает позицию всей группы.
- **Пазл/Тетрис:** один элемент переставляется, и общая комбинация становится лучше.
- **Мост/карта метро:** новая книга создаёт связь между участниками и сценариями.
- **Расстановка за столом:** все уже сидят, но можно пересадить так, чтобы разговоры стали лучше.

Рабочая формулировка: “Добавь эту книгу — откроется более удачный расклад”.
