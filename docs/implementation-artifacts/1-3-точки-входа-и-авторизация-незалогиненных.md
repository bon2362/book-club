# Story 1.3: Точки входа и авторизация незалогиненных

Status: done

## Story

As any user (authenticated or not),
I want to access the book submission form from the header and the catalog grid,
So that I can easily discover and initiate a book submission from anywhere on the site.

## Acceptance Criteria

1. **[AC1 — Псевдо-карточка]** Псевдо-карточка "Предложить книгу" отображается первой в сетке книг (в grid-режиме)

2. **[AC2 — Кнопка в хедере]** Кнопка "Предложить книгу" видна в хедере для всех пользователей

3. **[AC3 — Прямой доступ]** Авторизованный пользователь кликает на карточку или кнопку в хедере → форма открывается напрямую (без попапа авторизации)

4. **[AC4 — Auth попап]** Неавторизованный пользователь кликает → отображается попап авторизации

5. **[AC5 — Auto-open после входа]** Неавторизованный пользователь завершил вход через попап → форма заявки открывается автоматически

## Dev Notes

### Архитектурный выбор: Context vs Props

Архитектура предлагала `SubmitBookIntentContext`. Однако все компоненты (`SubmitBookCard`, `SubmitBookButton` в Header, `SubmitBookForm`) рендерятся внутри `BooksPage` — Header является дочерним компонентом BooksPage, который передаёт ему колбэки через props:

```typescript
<Header
  onEditProfile={...}
  onSignIn={...}
  onSubmitBook={handleSubmitBookClick}   // добавить
/>
```

Поэтому контекст не нужен — используем prop drilling. Это соответствует принципу "минимальной сложности" для текущей задачи.

- Содержимое по центру: иконка + заголовок + подпись

### Intent flow (post-auth auto-open)

```typescript
// В BooksPage:
const [intent, setIntent] = useState<null | 'submit-book'>(null)

useEffect(() => {
  if (isLoggedIn && intent === 'submit-book') {
    setSubmitFormOpen(true)
    setIntent(null)
  }
}, [isLoggedIn, intent])

function handleSubmitBookClick() {
  if (isLoggedIn) {
    setSubmitFormOpen(true)
  } else {
    setIntent('submit-book')
    setAuthModalOpen(true)
  }
}
```

Работает для Telegram auth (in-page, `router.refresh()` сохраняет React-state).
Для Google OAuth (redirect) intent теряется — приемлемо для MVP согласно архитектуре.
