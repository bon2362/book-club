## Что хорошо сработало

1. **Правильный порядок историй.** DB/API → Форма → Точки входа. Каждая история строилась на готовом фундаменте, переделок не было.

2. **Полное покрытие тестами формы (Story 1.2).** 11 тестов: render, validation (onBlur + submit), success state, error state, close — все пользовательские пути покрыты.

3. **`submitIntent` паттерн (Story 1.3).** Элегантное решение для авто-открытия формы после авторизации: локальный стейт в BooksPage без глобального контекста, `useEffect([isLoggedIn, submitIntent])` — работает надёжно.

4. **Модальный success state.** Подтверждение отправки внутри модалки — правильное UX-решение, пользователь не теряет контекст страницы.

5. **Aria-атрибуты.** `role="dialog"`, `aria-label` на кнопках входа — доступность учтена с первого раза.

---

## Что можно улучшить

1. **Нет E2E-теста для критического пути** (незалогиненный → авторизация → автооткрытие формы). Unit-тесты есть, но сквозной сценарий не покрыт автоматически.

2. **Мобильная доступность не верифицирована автоматически.** NFR1-4 (mobile UX) задокументированы в требованиях, но нет тестов на mobile viewport.

---

## Ключевые инсайты

1. Разбивка на маленькие истории с чёткими зависимостями → 0 блокеров между историями.
2. Тесты с `@testing-library/react` хорошо проверяют поведение формы; стоит использовать этот паттерн во всех UI-компонентах.
3. Intent-паттерн (сохранить намерение, выполнить после условия) — переиспользуемый подход для аналогичных сценариев.
