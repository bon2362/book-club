## Описание обработки

### 1. Учёт участников клуба

| Поле | Значение |
|---|---|
| Категории субъектов | Зарегистрированные пользователи slowreading.club |
| Категории данных | Email, имя, Telegram-username, список выбранных книг, отметки администраторов |
| Цели | Связь с участниками, формирование подборок, организация встреч |
| Правовое основание | Законный интерес контроллера + добровольное согласие через регистрацию |
| Получатели | Neon (Postgres, US/EU), Resend (для писем), Google и Telegram (для авторизации) |
| Срок хранения | Пока активный аккаунт; немедленное удаление по запросу |
| Технические меры | HTTPS, изолированная БД, доступ только с server-side через auth |

### 2. Продуктовая аналитика

| Поле | Значение |
|---|---|
| Категории субъектов | Все посетители сайта |
| Категории данных | Внутренний UUID, страницы, действия, тип устройства/браузера, страна по IP |
| **Не собираются** | Email, имя, точный IP (PostHog маскирует), точное местоположение |
| Цели | Понимание использования сайта, улучшение продукта |
| Правовое основание | Законный интерес (legitimni interes) |
| Получатели | PostHog Inc. (через PostHog Cloud EU, Финляндия) |
| Срок хранения | До 7 лет (default PostHog), может быть сокращён |
| Технические меры | EU-регион, identify только UUID без PII, respect_dnt включён, billing limits в $0 для защиты от шумовых атак, автоматическое удаление PostHog-профиля при удалении аккаунта |

## Уведомление об утечке

При обнаружении инцидента, повлёкшего раскрытие персональных данных:
1. Зафиксировать факт и охват
2. В течение 72 часов уведомить Поверenikа za informacije od javnog značaja i zaštitu podataka o ličnosti
3. Если есть высокий риск для субъектов — уведомить пострадавших по email

Поскольку в PostHog передаются только обезличенные UUID, утечка PostHog в подавляющем большинстве случаев **не** требует индивидуального уведомления пользователей — связки UUID → личность нет вне нашей БД.
