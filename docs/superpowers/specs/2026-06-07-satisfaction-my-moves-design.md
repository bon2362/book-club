# Пояснения к «Моим ходам» в satisfaction-режиме

**Issue:** [#316](https://github.com/bon2362/book-club/issues/316)
**Дата:** 2026-06-07
**Область:** матчинг-сессия, карточки «Мои ходы», режим `satisfaction`

## Проблема

В satisfaction-режиме карточка «Мои ходы» объясняет выгоду хода технически и
непонятно:

- пилюля показывает `↑ ранг 20→1` — жаргон («ранг») и цифра *собственного* ранга
  зрителя, хотя реальная ценность хода в этом режиме почти всегда **групповая**;
- why-text — общая отписка: «Этот ход соберёт круг, где интересы совпадают лучше.
  Остальные участники показаны как контекст».

Корень: satisfaction-режим оптимизирует `worstRank` **всего сценария** (самого
недовольного участника в раскладе — [`compareScenarioSatisfaction`](../../../lib/matching/scenarios.ts)),
а карточка рассказывает про ранг зрителя. Из-за этого ходы, которые улучшают
расклад для *другого* участника (но не для зрителя), либо не показываются, либо
показываются с непонятной формулировкой.

Пример из issue: Нарвал добавляет «Заря всего». Барсук сейчас в круге книги, которую
он поставил на 20-е место; «Заря всего» у Барсука на 1-м. Собравшись вокруг неё,
Барсук переезжает с 20-го на 1-е — расклад становится лучше. Карточка про это молчит.

## Решения, принятые на брейншторме

1. **Что показывать.** Показывать *любой* ход, улучшающий расклад, даже если по
   удовлетворённости у самого зрителя ничего не меняется (альтруистичные ходы).
   Ходы, которые оставили бы зрителя за бортом, не показывать — это уже гарантирует
   `scenarioIncludesMove` ([`app/matching/page.tsx`](../../../app/matching/page.tsx)),
   который требует, чтобы зритель был в круге добавленной книги.
2. **Формулировка выгоды.** Комбинация: тёплый хедлайн-пилюля + конкретные места в
   why-text. Слово — «интереснее», не «ближе».
3. **Склонение псевдонимов.** Подключаем библиотеку склонений `russian-nouns-js`
   (dev-генератор), для редких/неверных зверей — ручные оверрайды. Склоняем
   свободно везде, включая why-text; целевая фраза-пилюля — «Барсуку станет
   заметно интереснее».
4. **Число beneficiary в why-text не ограничиваем** — показываем всех (владелец
   протестирует на проде и решит, надо ли резать).

## Архитектура изменений

### 1. Склонение псевдонимов (новый модуль + генератор)

**Цель:** статичная таблица форм псевдонимов, без зависимости от библиотеки в
рантайме (паттерн как у `lib/matching/species-images.generated.ts`).

- `scripts/generate-pseudonym-declensions.ts` — dev-генератор: берёт `ANIMALS` из
  `lib/matching/pseudonyms.ts` + карту рода, прогоняет через `russian-nouns-js`,
  пишет `lib/matching/pseudonym-declensions.generated.ts` со всеми падежами
  (NOM/GEN/DAT/ACC/INS/PREP) для каждого зверя.
- Карта рода `PSEUDONYM_GENDER: Record<string, 'm'|'f'|'n'>` — нужна библиотеке на
  входе; держим в исходнике генератора (или отдельном `pseudonym-gender.ts`),
  выверяется вручную.
- Ручные оверрайды `PSEUDONYM_DECLENSION_OVERRIDES` — мерджатся поверх
  сгенерированного (для зверей, где библиотека ошиблась). Хранятся в hand-файле,
  не перезаписываются генератором.
- `russian-nouns-js` добавляется в `devDependencies`.
- **Перед слиянием:** вывести владельцу полный список `зверь → род → дательный`
  (и спорные падежи) для ревью; внести правки в карту рода / оверрайды.

**Рантайм-API** (`lib/matching/pseudonym-declension.ts`):
- `declinePseudonym(name: string, c: Case): string` — читает статичную таблицу,
  фолбэк на именительный (исходный `name`), если зверя нет.
- `pseudonymPronoun(name, form)` — местоимения из рода (`он/она/оно`, `его/её`,
  `ему/ей`, `него/неё`). Маленькая фиксированная таблица по роду, данных на каждого
  зверя не требует.
- Удобные шорткаты по необходимости (`pseudonymDative(name)`).

### 2. Логика отбора и данные — `lib/matching/move-impact.ts`

**Beneficiary по рангу (не по тиру интереса).** Текущий фильтр использует
`INTEREST_TIER` и пропускает улучшения внутри одного тира (8→5 — оба «хочу»). Для
satisfaction считаем beneficiary по числовому рангу:
- участник круга добавляемой книги (кроме зрителя), чей `afterRank` (ранг по
  добавляемой книге) **строго меньше** его `before.rank` (ранг в прежней книге
  круга), либо кто раньше был `leftOut` / без ранга.

**Данные.** Дополнить `MyMoveBook.impact.beneficiaries`:
- в `before` circle-вариант добавить `rank: number | null`;
- добавить `afterRank: number | null`.
Для этого `placeBefore` должен хранить ранг участника в прежнем круге (берётся из
`currentLeader.circles[].members[].rank`).

**Отбор хода (satisfaction-ветка).** Сейчас: `if (!wasLeftOut && !improvedRank)
return null`. Станет — показать ход, если выполнено **любое**:
- есть хотя бы один satisfaction-beneficiary (другой участник, чей ранг улучшился
  или кто вышел из-за борта/без ранга), **или**
- улучшился ранг самого зрителя (`improvedRank`), **или** зритель вышел из-за борта
  (`wasLeftOut`).
Если ни одного — `return null` (ход ничего не меняет).

### 3. Сортировка — `sortMovesByImpact` (satisfaction)

Сейчас сортирует по выигрышу ранга *зрителя*; альтруистичные ходы (ранг зрителя не
меняется) проваливались бы в `-Infinity`. Станет — сортировать по **максимальному
улучшению ранга среди показанных beneficiary** (включая зрителя): для каждого
beneficiary считаем `before.rank - afterRank` (для вышедших из-за борта — максимум),
берём лучший по ходу, сортируем убыванием; тай-брейк — название.

### 4. Копирайтинг — `components/nd/MatchingMyMoves.tsx`

**Пилюля** (`ImpactMetricPills`, satisfaction-ветка) вместо `↑ ранг 20→1`:
- другим участникам станет лучше → **«{Имя(дат)} станет заметно интереснее»**
  (1 чел); несколько → «{Имя1(дат)} и {Имя2(дат)} — интереснее»;
- выигрывает только зритель → «тебе — интереснее»;
- кто-то выходит из-за борта/без ранга → «соберётся круг».

**Why-text** (`MoveWhyText`, satisfaction-ветка) — конкретные места со склонением:
- 1 beneficiary: «{Имя(дат)} станет заметно интереснее: твою книгу {он/она} ставит
  на {afterRank}-е место, а нынешнюю книгу круга — на {beforeRank}-е. Соберётесь
  вокруг неё — расклад станет интереснее.»
- несколько: перечислить, имена в именительном для однозначности
  («Барсук ставит твою книгу на 1-е место, Журавль — на 3-е…»), хвост — «соберётесь
  вокруг неё, расклад станет интереснее».
- вышедшие из-за борта/без ранга: «{Имена} сейчас без круга — добавишь, и соберётесь
  вместе».
Точные строки финализируются при имплементации; владелец проверит тон на проде.

## Вне области (YAGNI)

- Не трогаем coverage-режим карточек.
- Не трогаем алгоритм скоринга сценариев (`compareScenarioSatisfaction`) — только
  отображение.

  выше.
