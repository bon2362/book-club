# Бэклог

Оценка сложности: **XS** < 1 ч · **S** полдня · **M** 1–2 дня · **L** 3+ дней

## Идеи для развития

### Процессы разработки

#### 32. Валидация env-переменных · `S`

`process.env.X` возвращает `undefined` без предупреждений — ошибка проявляется глубоко в рантайме. Добавить `@t3-oss/env-nextjs` или zod-схему: при отсутствии обязательной переменной сервер падает на старте с понятным сообщением.

#### 70. Система бэкапов · `M`

Автоматические резервные копии базы данных (Neon Postgres). Варианты: встроенные бэкапы Neon (Point-in-Time Recovery, доступен на платных планах), либо cron-задача через GitHub Actions — pg_dump в зашифрованный архив в S3/R2/GitHub Releases. Определить частоту (ежедневно?), срок хранения и процедуру восстановления.

#### 34. Мониторинг ошибок (Sentry) · `S`

Продакшен-ошибки видны только если пользователь сообщит. Подключить Sentry (`@sentry/nextjs`) — бесплатный tier достаточен. Даёт трассировку, контекст запроса и алерты на email при необработанных ошибках.

### UI/UX

#### 64. Контекстные подсказки по мере использования сайта · `M`

После ключевых действий показывать пользователю короткие ситуативные подсказки — чтобы объяснить, что происходит дальше, и помочь освоить возможности сайта.

**Примеры подсказок:**
- После первой записи на книгу: «Отлично! Ты записал:ась на книгу — как наберётся достаточно народа, мы соберём вас в общую телеграм-группу»
- После записи на вторую книгу: «Кстати, в личном кабинете можно расставить книги по приоритету — это поможет подобрать тебе подходящую группу»

**Формат — открытый вопрос.** Варианты: toast снизу с задержкой (как нынешние уведомления, но с другим стилем), inline-блок под карточкой книги, баннер в drawer'е, отдельная «подсказка дня» под шапкой. Хранить факт просмотра в `localStorage`, чтобы не показывать повторно.

**Возможные триггеры:**
- Первая запись на книгу
- Вторая запись на книгу (подсказка о приоритетах)
- Первый открытый личный кабинет
- Заполнение профиля (имя/контакты)

#### 22. Кнопка переключения тёмной темы · `M`

Тёмный режим работает через `prefers-color-scheme`, но пользователь не может переключить его вручную. Добавить кнопку в хедер (иконка солнца/луны), сохранять выбор в `localStorage`.

#### 23. Toast-уведомления об ошибках · `S`

Многие ошибки API молча проглатываются или меняют текст кнопки — пользователь может не понять, что что-то пошло не так. Добавить простой toast-компонент (fixed div снизу экрана) для отображения ошибок и успешных действий.

#### 24. Пагинация или виртуализация списка книг · `M`

Все книги загружаются сразу. При 50+ книгах на мобильном это заметно. Добавить простую пагинацию или `react-virtual` для виртуализации длинного списка.

#### 66. Email в профиле — не кликабельная ссылка · `XS`

В личном кабинете (вкладка «Профиль») email пользователя отображается как кликабельный `mailto:`-линк. Сделать его некликабельным — просто текстом, без `<a>`.

#### 67. Имя в заголовке профиля из поля «Имя» · `XS`

Заголовок личного кабинета использует email или другое поле для отображения пользователя. Если у пользователя заполнено поле «Имя» — показывать имя; если нет — откатываться к текущему поведению (email).

#### 65. Форма обратной связи · `S`

Добавить способ для участников отправить обратную связь — вопрос, предложение или пожелание. Форма с полем текста, без обязательной авторизации (но для залогиненных — имя подставляется автоматически). Сообщения отправляются на `hello@slowreading.club` через Resend. Разместить в подвале или в виде кнопки «Написать нам» в шапке.

### Авторизация

#### 68. Google One Tap · `S`

Автоматический всплывающий промпт от Google при загрузке главной страницы — незалогиненный пользователь с аккаунтом Google в браузере может войти одним кликом, без открытия AuthModal. Новый Credentials provider в NextAuth, верификация JWT через `google-auth-library`, создание записей в `users` + `accounts` для новых пользователей.

[План реализации →](docs/superpowers/plans/2026-03-18-google-one-tap.md)

#### 71. Выделение топ-3 в списке «Записал:ась» · `XS`

В вкладке «Записал:ась» личного кабинета первые три места уже отмечены оранжевыми кружками с номером. Идея: сделать разграничение более явным — например, добавить эмодзи-медали (🥇🥈🥉) рядом с номером или отделить топ-3 визуальной чертой от остального списка.

#### 72. Заглушки вместо цифр до первой расстановки · `XS`

До тех пор пока пользователь не переставил хотя бы одну книгу (флаг `priorities_set = false`), вместо порядковых номеров `1`, `2`, `3`... отображать условные знаки — например `?` или `—`. Это подчёркивает, что порядок ещё не задан и мотивирует его расставить.

#### 73. Счётчик «на первом месте» на карточке книги · `S`

В каталоге книг (или во вкладке «По книгам» admin-панели) рядом с общим числом записавшихся показывать, у скольких из них эта книга стоит на **первом месте** в рейтинге. Данные из таблицы `book_priorities`. Помогает оценить реальный интерес: не просто «записался», а «очень хочу».

#### 74. Группировка email-уведомлений о новых записях · `S`

Сейчас при каждой новой записи на книгу организатору приходит отдельный email. Идея: накапливать уведомления и отправлять дайджест — например, раз в час или раз в день. Если за период записалось несколько человек, приходит одно письмо со списком. Варианты реализации: cron-задача (Vercel Cron или GitHub Actions) + очередь в Postgres/KV, либо debounce на уровне API с отложенной отправкой через Resend Scheduled Emails.

#### 4. Вход через Telegram · `L`

Добавить авторизацию через Telegram (Telegram Login Widget) как альтернативу Google OAuth и email magic link. Если пользователь вошёл через Telegram — не запрашивать Telegram-контакт отдельно, а подставлять его автоматически из данных авторизации и блокировать поле в ContactsForm.
**Статус:** не работает — Telegram не доставляет код авторизации в приложение. Виджет и конфигурация бота корректны, проблема на стороне Telegram OAuth. Оставлено до выяснения.

### Функциональность

#### 42. Разбивка участников по группам (admin) · `L`

Администратор выбирает, как разбить участников, записавшихся на одну книгу, по группам для обсуждения. UI в admin-панели: список книг → участники → ручное распределение по группам (drag-and-drop или выпадающий список). Результат сохраняется и, опционально, рассылается участникам группы по email.

#### 47. Саммари книги от участников · `L`

Авторизованный участник может написать короткое саммари (личные впечатления, выводы) по книге, которую клуб уже прочитал. Саммари привязано к книге и пользователю, отображается на странице книги или в карточке. Несколько участников могут оставить саммари по одной книге.

Затрагивает: новая таблица в `lib/db/schema.ts` (book_summaries: bookId, userId, text, createdAt), новый API-эндпоинт, UI на карточке/странице книги.

### 41. Ранжированное голосование за книги

Drag-and-drop расстановка приоритетов в личном кабинете (вкладка «Записал:ась»). Порядок книг сохраняется в таблице `book_priorities` (Neon Postgres) через `PUT /api/priorities`. Топ-3 отмечены оранжевыми кружками. Баннер-подсказка скрывается после первой сортировки (`priorities_set` флаг в таблице `users`). Автосохранение с debounce 500 мс, индикатор «✓ Сохранено». При отписке от книги записи приоритетов очищаются автоматически. Администратор в таблице участников может фильтровать по книге и видит колонку «Приоритет» с бейджем `№X из N`; участники без приоритетов вынесены в отдельную группу.

Новые файлы: `app/api/priorities/route.ts`, `app/api/admin/priorities/route.ts`, `drizzle/0004_uneven_grandmaster.sql`. Изменены: `lib/db/schema.ts`, `app/api/signup/route.ts`, `components/nd/ProfileDrawer.tsx`, `components/nd/AdminPanel.tsx`.

### 61. Отображение приоритетов в admin-панели

Вкладка **Участники**: бейджи книг показывают приоритет пользователя — чёрный префикс `#1`, `#2`… для расставленных книг, серый `+` для книг добавленных после расстановки, серый `?` с подписью «Приоритеты не расставлены» для пользователей без приоритетов. Книги отсортированы по рангу. Фильтр по книге удалён (больше не нужен). Вкладка **По книгам**: рядом с именем каждого участника показывается его приоритет `(#N)`, отсортированы по рангу. При удалении книги у пользователя через `×` — приоритеты автоматически пересчитываются в БД (re-rank) и в UI без перезагрузки.

Изменены: `app/api/admin/remove-book/route.ts`, `app/admin/page.tsx`, `components/nd/AdminPanel.tsx`.

Показывать на карточке книги (или в другом месте сайта), сколько человек записалось на каждую книгу. Данные брать из Google Sheets (таблица записей). Помогает участникам видеть популярность книг и принимать решение о записи.

### 13. Снятие пользователя с книги администратором

В панели администратора на вкладке «Участники» рядом с каждой книгой в столбце «Книги» добавить кнопку-крестик. По нажатию — диалог подтверждения с именем пользователя, затем удаление книги из записи пользователя в Google Sheets. Оптимистичное обновление UI. Если у пользователя не осталось книг — запись остаётся (удаление пользователей — отдельная фича).

### 16. Вход через magic link

Email-поле + кнопка «Получить ссылку на почту» в `AuthModal` под кнопкой Google. Письма отправляются через Resend с `noreply@slowreading.club`. Кастомный HTML-шаблон с брендингом клуба, plain text fallback, тема «Ссылка для входа в книжный клуб». Домен верифицирован в Resend и Google Postmaster Tools.

### 15. Переключение вида списка книг: сетка / таблица

Иконки-переключатели в строке фильтров (справа). По умолчанию — сетка карточек. Табличный вид: компактные строки без обложек и описаний, цветная полоска статуса слева, название + год + автор, теги, страницы + ссылка, счётчик записавшихся, кнопка действия. Hover-подсветка строк. Выбор сохраняется в `localStorage`.

### 14. Удаление пользователя администратором

Кнопка «Удалить» в таблице участников с диалогом подтверждения. Удаляет пользователя из Postgres (каскадно accounts/sessions), помечает строку в Sheets G=TO DELETE + H=yes (is_deleted_by_admin). JWT callback проверяет существование пользователя в DB — при отсутствии возвращает null, что очищает cookie и разлогинивает пользователя на следующем запросе.

### 54. Личный кабинет пользователя

Drawer справа, открывается по клику на имя пользователя в шапке. Три вкладки:

- **Записал:ась** — книги, на которые пользователь записан; кнопка × для отписки с возможностью отмены («↩ вернуть») до перезагрузки страницы

- **Профиль** — форма имени и Telegram (кнопка «Сохранить» активна только при изменениях), языковые предпочтения с auto-save, выход из аккаунта

Гендергэпы («Записал:ась», «Предложил:а») во всём интерфейсе drawer'а и в фильтрах.

Toast-уведомления об успехе и ошибках для всех действий.
