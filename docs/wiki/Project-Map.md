# Карта проекта

Эта страница отвечает на вопрос: из каких больших частей состоит проект и за что каждая часть отвечает.

## Основные пользовательские зоны

| Зона | Для кого | Что делает |
| --- | --- | --- |
| Главная страница | Посетители и участники | Показывает каталог, фильтры, intro-блок, форму входа и кнопку предложения книги. |
| Профиль | Авторизованные участники | Хранит имя, контакт, языки чтения, выбранные книги и порядок приоритетов. |
| Саммари книг | Авторизованные участники + Admin | Участники пишут Markdown-саммари по прочитанным книгам, администратор модерирует и публикует. |
| Предложение книги | Авторизованные участники | Позволяет предложить книгу или текст в каталог. |
| Матчинг | Авторизованные участники + Admin | Координационное пространство для выбора читательских групп. Личный список с drag-and-drop, сценарии групп, «Мои ходы», realtime через polling версии сессии. |
| Админка | Владелец проекта | Управляет участниками, каталогом, тегами, заявками, фидбеками, intro-блоком и matching-сессиями. |
| API docs | Владелец и разработчик | Показывает список доступных API-эндпоинтов и их контракты. |

## Главные технические зоны

| Зона | Где живет | Зачем нужна |
| --- | --- | --- |
| Next.js app | Vercel | Рендерит сайт, страницы и API. |
| Neon Postgres | Neon | Хранит пользователей, книги, записи, заявки, фидбек, активности и очередь уведомлений. |
| NextAuth | Внутри приложения | Управляет входом через Google, email magic link, Google One Tap и Telegram. |
| Resend | Внешний сервис | Отправляет magic link и email-уведомления. |
| GitHub Actions | GitHub | Запускает CI, E2E, Allure-публикацию и digest-cron. |
| GitHub Wiki | GitHub | Хранит эту документацию. |
| GitHub Pages | GitHub | Публикует Allure-отчеты по E2E. |
| Codecov | Внешний сервис | Показывает покрытие unit-тестами. |
| PostHog | Внешний сервис | Считает продуктовую аналитику и месячное использование событий. |

## Что больше не является актуальной архитектурой

Раньше проект сильно опирался на Google Sheets. Сейчас это больше не runtime-источник каталога или записей. Google Sheets может встречаться в старых планах, прототипах и ретроспективах, но актуальная модель живет в Postgres.
