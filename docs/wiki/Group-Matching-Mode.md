# Матчинг читательских кругов

Матчинг работает в одном режиме: он ранжирует возможные круги по удовлетворённости участников. Режима coverage, псевдонимов, секций «Мои ходы» и «Лента событий» больше нет; legacy-поля и таблицы также удалены из production-схемы. Старую реализацию можно восстановить по git-тегу `matching-legacy-before-simplification-2026-06-29`.

## Пользовательский путь

0. Неавторизованный пользователь, открывший `/matching`, не редиректится на главную, а видит стандартное окно входа поверх страницы. После входа (Telegram/Google/Email) он остаётся на `/matching` и продолжает обычным путём с Welcome. Закрытие окна уводит на главную.
1. При первом входе в конкретную сессию Welcome объясняет, что настоящее имя будет видно всем участникам страницы. Имя можно исправить прямо здесь; меняется глобальное `user.name`.
2. Ranking Gate («Сначала — расставь приоритеты») — обязательный порог перед доской: на неё пускают только тех, у кого есть хотя бы одна активная книга (в списке, не «в процессе/прочитано») со всеми расставленными рангами. Пока этого нет, показывается Gate. Если активных книг нет вообще (пусто или только «в процессе/прочитано»), Gate адаптирует текст на «Выбери книги для клуба» и просит добавить книгу из каталога. Кнопка «Войти в сессию» активна, как только есть хотя бы одна книга, и сама фиксирует текущий порядок при клике (commit-on-enter) — поэтому случай с единственной книгой, где перетаскивать нечего, тоже работает. Книга участвует в расчёте только после назначения ранга.
3. На основной доске показаны равноправные ранжированные сценарии. Первый вариант не выделяется как «лучший».
4. В круге, куда входит пользователь, можно нажать «Хочу в этот круг». До единогласия выбор временный и отменяемый; подтверждённых участников видят все, у кого есть доступ к странице.
5. Когда точный состав подтвердили все, круг закрепляется. Участники становятся наблюдателями, исключаются из дальнейшего расчёта, но продолжают видеть страницу и оставшиеся сценарии.

Telegram-кнопок в matching нет: клуб уже общается в существующем канале.

В шапке остаются возврат в каталог, название и состояние сессии, размеры кругов, дедлайн, имя/роль пользователя и полный список участников с online-индикаторами. В сценариях показываются обложки, авторы, ранги участников, средний ранг, охват и люди вне кругов; обложка или название открывают тот же popup книги, что и каталог.

## Что именно подтверждается

Подтверждение относится к конкретной книге и точному составу людей (`circle_key`). У пользователя может быть только одно активное подтверждение.

- Пока выбранный точный круг существует, подтверждение остаётся на нём независимо от позиции в сценариях.
- При ручном выборе другого круга интерфейс предупреждает о замене; сервер выполняет перенос атомарно.
- Если выбранный круг распался, но для той же книги есть другой круг с этим участником, подтверждение переносится автоматически. При нескольких вариантах выбирается круг из наиболее высоко ранжированного сценария.
- Если кругов с выбранной книгой больше нет, подтверждение сбрасывается.

Перенос сразу участвует в проверке единогласия. Пользователь получает durable-уведомление о переносе или сбросе.

## Закреплённые круги

Закрепление необратимо для участников. Над живыми сценариями показывается реестр закреплённых кругов. Администратор имеет отдельное аварийное действие «Распустить круг»: требуется непустая причина, освобождается весь круг целиком, а его участники возвращаются в активный расчёт. Частичный роспуск невозможен.

После каждого закрепления система пересчитывает оставшихся участников, переносит устаревшие подтверждения и автоматически закрепляет новые единогласные круги, пока состояние не станет устойчивым.

## Расчёт и обновление страницы

В расчёт входят только:

- участники роли `active`;
- активные записи на книги;
- книги с назначенным рангом.

Сценарии сортируются лексикографически. Сначала круги каждого сценария упорядочиваются по меньшему среднему рангу, меньшему худшему рангу, большему составу и стабильному `id`; затем два сценария сравниваются круг за кругом. Если круги равны, учитываются меньший общий средний ранг, большее число сильных интересов (ранг 1–3) и стабильный `id` сценария.

Генератор намеренно ограничивает пространство поиска ради предсказуемого времени ответа: не более 24 разнообразных candidate-кругов на книгу, не более 900 candidate-кругов суммарно и beam шириной 50 состояний на каждом шаге. Это технические пределы поиска, а не UI-лимит количества уже найденных сценариев.

Клиент опрашивает `/api/matching/version`. Heartbeat обновляет только признак присутствия и список online; когда `state_version` меняется, клиент получает полный `/api/matching/state`. WebSocket и мгновенное обновление не требуются, поэтому изменения другого участника гарантированно видны после polling или перезагрузки.

Heartbeat не является решением или изменением предпочтений. Его чистые обновления `last_seen_at` специально исключены и из «Аналитики изменений предпочтений», и из глобальной «Истории изменений», чтобы журналы не превращались в поток технического шума.

## Данные и приватность

Обычный public state содержит реальные `displayName`, но не raw `userId`. Для связей используются непрозрачные `publicRef` и `circleKey`. Одинаковые имена разрешены; при коллизии интерфейс стабильно показывает локальные номера, например `Анна (1)` и `Анна (2)`.

Основные таблицы:

- `matching_sessions` — сессия, размер групп, статус и версия;
- `matching_session_participants` — участники, `public_ref`, `join_source`, presence;
- `matching_circle_confirmations` — одно временное подтверждение на пользователя;
- `matching_locked_circles` и `matching_locked_circle_members` — закреплённые и распущенные круги;
- `matching_notices` — непрочитанные персональные уведомления;
- `matching_events` — смысловая история для админской аналитики.

Все доменные мутации выполняет `runMatchingTransition` в одной транзакции с `withAuditContext`: блокировка сессии, изменение данных, пересчёт, перенос подтверждений, каскад закреплений, events/notices и увеличение версии либо фиксируются вместе, либо полностью откатываются.

## Администрирование и журналы

Во вкладке «Матчинг» администратор может:

- создать и зафиксировать сессию, изменить диапазон размера групп;
- принудительно добавить участника для тестирования (это явно обходит Welcome disclosure);
- видеть реальные имена, источник входа, online и роль active/observer;
- просматривать закреплённые круги и аварийно распускать целый круг;
- заморозить сессию с сохранением snapshot результата;
- читать «Аналитику изменений предпочтений» из `matching_events`.

Те же мутации отражаются в глобальном `audit_log`. `matching_events` отвечает на продуктовый вопрос «что произошло в matching», а `audit_log` хранит технические before/after строк БД.

## Основные API

| Метод | Путь | Назначение |
| --- | --- | --- |
| GET | `/api/matching/state?session=...` | Полный public state участника или наблюдателя. |
| GET | `/api/matching/version?session=...` | Версия, статус и online refs для polling. |
| POST | `/api/matching/sessions/{id}/join` | Welcome: атомарно сохранить глобальное имя и вступить. |
| DELETE | `/api/matching/sessions/{id}/leave` | Покинуть сессию до закрепления круга. |
| PUT/DELETE | `/api/matching/sessions/{id}/confirmation` | Создать/переключить или отменить подтверждение. |
| POST/DELETE | `/api/matching/books`, `/api/matching/books/{bookId}` | Добавить или убрать книгу с транзакционным пересчётом. |
| PATCH | `/api/matching/priorities` | Сохранить порядок книг и пересчитать круги. |
| POST | `/api/matching/notices/{id}/ack` | Пометить персональное уведомление прочитанным. |
| GET | `/api/admin/matching/sessions/{id}/locked-circles` | Прочитать историю закреплённых и распущенных кругов с snapshots состава. |
| POST | `/api/admin/matching/sessions/{id}/circles/{circleId}/dissolve` | Аварийно распустить весь круг с причиной. |
| GET/POST | `/api/admin/matching/sessions/{id}/participants` | Список ролей/online или принудительное добавление участника. |
| DELETE | `/api/admin/matching/sessions/{id}/participants/{userId}` | Удалить активного участника. |
| POST | `/api/matching/sessions/{id}/freeze` | Администратору: заморозить сессию и snapshot. |
| GET | `/api/admin/matching/preference-events` | Админская аналитика из `matching_events` (URL сохранён для совместимости UI). |
